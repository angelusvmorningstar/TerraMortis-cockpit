## What went well

1. **The dev→QA separation earned its keep — empirically.** Independent adversarial QA
   (Quinn) caught a real, shippable privacy bug (18 haven addresses bound for an external
   AI) that the dev pass missed, plus genuine methodological softness in the first
   re-audit. On a solo project the temptation is to collapse dev and QA into one pass;
   this phase is the concrete argument against doing so.
2. **Structural blindness beat instructed blindness.** Rev 2 placed the drafters in a
   sandbox that physically could not reach the audit answer key; all nine disclosed
   pack-only `FILES READ`. Blindness became a property of the setup, not a promise.
3. **Grading got more honest under challenge.** Instance 1 graded down to "stated-the-gap
   (partial)", instance 13 reclassified, headline restated against an acknowledged-
   incomplete baseline. The measurement stopped flattering itself.
4. **Validation produced product, not just a score.** The re-audit yielded four concrete
   pack fixes (Story 2.8) and a clean Phase-2 scope list.
5. **Deliberate model routing.** Opus for the grading/re-audit (judgement, fabrication
   risk); Sonnet for the mechanical 2.7/2.8 fixes and their QA. Cost matched to the work.

## Challenges and lessons (no blame — systems and process)

1. **The NFR2 leak should have been caught in DEV, not QA.** The Character Index
   projection (Story 2.2) passed dossier fields through wholesale; no privacy check existed
   in the dev definition-of-done. QA is the backstop, not the first line.
   → Action C1.
2. **The first re-audit overclaimed and self-graded generously** — predictable, because
   the same activity drafted and measured. The adversarial split and honest-baseline
   framing should have been in the method from the start. → Action C2.
3. **Blindness-by-instruction is not blindness.** "Don't read X" beside an answer key one
   directory away is not isolation; enforce it structurally. → Action C3.
4. **Reopening a "done" epic is healthy, not a failure.** A later story's QA exposed a
   defect in an earlier story; the status system absorbed the reopen cleanly. "Done" is
   revocable when downstream work finds an upstream defect.
